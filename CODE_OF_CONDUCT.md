# Code of Conduct

## Our pledge

We want this project to be a respectful and constructive space for
discussion and collaboration.

Participants are expected to communicate with care, assume good intent
where reasonable, and focus criticism on ideas, code, documentation, and
design decisions rather than individuals.

## Expected behavior

Examples of behavior that contributes to a positive environment include:

- using welcoming and inclusive language
- being respectful of differing viewpoints and experiences
- accepting constructive feedback gracefully
- keeping discussions focused on the project
- giving proper attribution to upstream projects and contributors
- being mindful that this is an unofficial project related to another
  open source community

## Unacceptable behavior

Examples of unacceptable behavior include:

- harassment, insults, or personal attacks
- discriminatory language or behavior
- deliberate intimidation or disruption
- publishing private information without permission
- implying endorsement by the Spring Security team or Spring project when
  no such endorsement exists
- removing or obscuring upstream attribution

## Enforcement

Project maintainers may remove comments, close issues, reject
contributions, or block participants when necessary to keep the project
safe and respectful.

## Attribution

This code of conduct is intentionally lightweight. If the project grows,
it may adopt a more formal community code of conduct.
